# Challenges Faced During Course

## 1. File Paths and Directories

**Status: Ongoing**

I struggled at times with navigating directories and making sure scripts were running from the correct location. This caused errors because I was sometimes running commands from the wrong directory, resulting in errors.

To help with this, I started using:

```bash
pwd
ls
cd
```

to confirm my current location and check what files and directories were available.

I also:

- Check directory structures before running scripts.
- Learned the importance of verifying that files and folders exist before running commands.

I think I have improved at troubleshooting directory problems, but I still need to build more confidence when working with unfamiliar environments and deployment setups.

---

## 2. Taking Time to Double-Check Work and Avoid Rushing

**Status: Ongoing**

Another challenge I have faced is learning not to rush through tasks just to complete them. Sometimes moving too quickly meant I missed small details, skipped important checks, or created problems that took longer to fix later.

To help with this, I have started to:

- Double-check configurations before running commands (for example, making sure I select the correct security group).
- Read error messages carefully instead of immediately trying random fixes.
- Use AI as a support tool when I do not understand an error, not just to solve the problem but also to explain why it happened.
- Verify that changes have worked before moving on to the next step.
- Spend more time understanding why something works rather than only focusing on getting the final result.
- Keep my documentation as up to date as possible so I can track my steps and identify where things may have gone wrong.

I think I have improved in this area because I now understand that taking extra time to check my work often saves time later. I'm still working on finding the right balance between being thorough and being efficient so I can continue making progress without falling behind.

---

## 3. Python Programming

**Status: Ongoing**

I found learning Python challenging at times, especially understanding how different concepts connect together and how to structure code effectively. Topics such as error handling, file handling, APIs, and testing required regular practice to fully understand.

To help improve my Python skills, I need to :

- Practise writing smaller programs to understand individual concepts.
- Use debugging and error messages to understand where problems were occurring.
- Break larger problems into smaller steps before writing code.
- Review examples and documentation when learning new concepts.
- Use AI to help generate additional practice exercises so I can continue building confidence through repetition.

I have become more comfortable writing and debugging Python code, but I still want to improve my confidence, problem-solving approach, and ability to write cleaner solutions independently.
